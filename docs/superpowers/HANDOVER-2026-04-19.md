## Plan 2.5 — do this BEFORE Plan 3

Surfaced during the manual smoke test after Plan 2 merged. Filed as [#16](https://github.com/buchbend/lore/issues/16).

**The problem.** All curator LLM call sites go through `anthropic.Anthropic().messages.create(...)` — billed per-token against an Anthropic API key. Users on a Claude Code (Pro / Max) subscription should NOT double-pay. The `anthropic_client` seam needs to pivot to a `claude -p "<prompt>"` subprocess-based backend.

**Why block Plan 3+:** each subsequent plan adds more LLM call sites (cluster-deeper, supersession judgment, etc.). Every new call site landed on the SDK path is one more place to convert later. Do the pivot once, at a low-surface-count moment.

**Affected modules:**
- `lib/lore_curator/noteworthy.py::classify_slice` — Curator A noteworthy filter
- `lib/lore_curator/session_filer.py::_merge_judgment` — Curator A merge-or-create
- `lib/lore_curator/cluster.py::cluster_session_notes` — Curator B clustering
- `lib/lore_curator/abstract.py::abstract_cluster` — Curator B extraction

**Design questions for the Plan 2.5 spec:**
- How does `claude -p` surface structured output? Verify current Claude Code CLI flags (`--output-format json` if it exists) vs. prompting for a JSON-fenced block + regex-parse.
- Introduce a routing seam: `LlmClient` protocol with `SDKClient` and `SubprocessClient` implementations. Config `anthropic_backend: subscription | api`, default `subscription`.
- Rewrite the fake-Anthropic test pattern → fake-`LlmClient` protocol pattern.
- Model-tier resolver: map `ModelTier.middle → ["claude", "-p", "--model", "claude-sonnet-4-6", ...]`.

**Shape:** probably 6–8 tasks. Small surface, big correctness win. Land it before picking up Plan 3.

## Gotchas worth remembering

Things that bit us during Plans 1–2. Future sessions should plan around these:

**Type / API shape surprises:**
- `read_attach(path)` returns a **flat dict** (`{"wiki": ..., "scope": ...}`), NOT an envelope `{"block": {...}}`. Early spec draft got this wrong; code is authoritative.
- `TranscriptLedger.advance()` accepts an **optional `curator_a_run: datetime | None` kwarg** — must be passed by the pipeline to keep the mtime re-trigger path alive. Issue #14 was filed and fixed for this.
- `claude-agent-sdk`: `list_sessions(directory=Path)` — keyword is `directory`, not `cwd`. `get_session_messages(session_id)` yields message dicts with `role` + `content` blocks (text / thinking / tool_use / tool_result).

**Architecture / layering:**
- `lore_core/*` must **not** import from `lore_cli/*` — that caused a latent circular import. `read_attach` (+ parser helpers) lives in `lore_core/attach.py`; `lore_cli/attach_cmd.py` re-exports for backward compat.
- `_REGISTRY` in `lib/lore_adapters/registry.py` must be declared **before** any code path that could call `register()` — ordering fragility flagged by final code review.

**Tooling / environment:**
- Editable install (`pip install -e <path>`) points at one path only. Switching worktrees breaks imports until you reinstall. Solution: after removing any worktree, always `pip install -e /home/buchbend/git/lore` from the main repo.
- `rich.Console.print()` does **not** accept `file=` kwarg. Pattern: `err_console = Console(stderr=True)` + `err_console.print(...)`. Issue #13 was filed and fixed for this.
- Claude Code plugin hooks are declared at `.claude-plugin/plugin.json` (NOT `hooks.yml` / `hooks.toml`). The repo's v1 shape: each event key is a list of `{matcher, hooks: [{type, command}]}` entries.

**Curator / hot path:**
- Hook extensions in `cmd_session_start` (or any other session hook) MUST be wrapped in try/except so a malformed ledger / unexpected error doesn't break the whole hook render.
- Anything that happens **after** `with curator_lock:` exits is racy with other curators. Plan 2's briefing publish intentionally releases the lock before publishing; tolerable because lock contention on a second concurrent run skips the earlier work.
- The "high-tier-off warning once per session" pattern uses a marker ID in `<lore_root>/.lore/warnings.log`. See `noteworthy.py::_emit_simple_tier_warning_once` + `abstract.py::_emit_high_off_warning_once` for the pattern to copy.

**Testing:**
- Fake Anthropic client pattern: `FakeAnthropic(responses_by_tool)` where keys are `tool_choice.name` strings. Curator B tests need keys for both `cluster` and `abstract`; Curator A tests need `classify` and `merge_judgment`.
- `from lore_adapters.registry import _REGISTRY` lets tests monkeypatch the registered claude-code adapter with a fake. Clean up via `monkeypatch` fixture or explicit pop.
- Settings allowlist in `.claude/settings.local.json` covers `Bash(pytest*)`, `Bash(git -C /home/buchbend/git/lore*)`, etc. Subagent execution doesn't need permission prompts for the standard operations.
