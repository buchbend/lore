## Gotchas the next session should know

1. **Plugin cache sync is manual.** If you `git reset` or your machine gets restored, you'll need to re-copy `.claude-plugin/plugin.json` from the source tree OR finally `git push` the v0.2.4 bump. Symptom: `hook-events.jsonl` stops growing.

2. **`lore runs list --hooks` shows pre-fix orphan runs.** The ~18 runs from 2026-04-20 with `trigger: manual, pending: 0` are ghosts from when the capture hook wasn't firing. They won't self-heal; they're historical records. Ignore them.

3. **`pending: 99` in `lore status` is per-vault, not per-wiki.** This is a P2 artefact — `TranscriptLedger.pending()` is global. A capture hook at `/home/buchbend/git/lore` adds all 97 transcripts from that cwd to the global ledger, inflating `pending` at every other scope's status view. P2 fixes this.

4. **Dangling-assertions edit mistake in test_session_filer.py.** When I added the Phase 1 test block, I used `old_string` matching only the first 4 lines of `test_slug_sanitises_title`, orphaning its last 2 assertions into my new test. Caught during red-check, fixed. Lesson: when adding tests next to an existing test function, match the WHOLE function body or append at end-of-file.

5. **`git push` is unpushed.** 4 local commits on `main`. The v0.2.4 is the one that practically matters — without it, fresh `claude plugin update lore@lore` invocations will still fetch the old manifest.

6. **GitNexus PostToolUse hook is noisy** ("GitNexus index is stale..."). Ignore it unless you're actively using GitNexus; it's not a lore concern.

7. **Review-pass recommendation was to drop P3**, not just defer it. If you land P5a and find merge behavior is actually fine, P3 can be permanently cut — the budget for its 40–60 lines of prompt tweaking is better spent on `lore session show <note>` (follow-up to P4) or starting v2.
